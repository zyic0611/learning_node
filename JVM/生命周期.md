# 生命周期

生命周期7阶段

1. 加载：通过类的全限定名获取二进制字节流;在内存中生成一个该类的java.lang.Class对象 作为访问入口
2. 连接
   - 验证：确保字节码流中的信息符合jvm规范
   - 准备: 为类变量（静态变量 分配内存并设置初始0值 （int 为0 引用为null 
   - 解析：将常量池内的符号引用替换为直接引用(内存地址) 比如字符串
3. 初始化:真正开始执行类定义中的java程序代码，为静态变量赋予代码中编写的实际初始值
4. 使用与卸载：类被程序使用 最终在没有任何引用时被gc回收。



## java关键字

1. static 静态 属于类
2. 普通对象 不带static 必须等人写了new 才会被分配在堆里
3. final 终态  赋值了无法改变 可以与static结合

### 时间维度

1. 准备阶段
   - 发生时机 代码刚被jvm读进内存  还没开始执行业务逻辑
   - 把所有static变量找出来 分配内存 并且赋予0值 int给0 对象null
   - **特权ConstantValue**：一个变量 满足 static+final+基本数据类型/String+编译期字面量 10/"abc" JVM会在这个阶段就赋值
2. 类的初始化阶段
   - 发生时机：准备阶段刚过 类马上要被使用
   - jvm开始执行类里的static赋值和static{}静态代码块 开始给普通静态变量真正赋值
   - 此时还是没有对象被new出来
3. 对象实例化阶段
   - 发生时机：业务代码里new
   - 去堆里拿内存 执行普通成员变量赋值 构造代码块{} 还有构造方法

### 举例

1. int a = 10 普通实例变量  在第三阶段对象实例化阶段 属于对象 不new 就没值 只能new的时候在堆里赋值
2. final int a =10  是不可变的普通实例变量 同1 在第三阶段 但是只能等new的时候在构造器里赋值 并且不可修改 final object a=new object() 也一样的
3. static int a =10 是普通类变量 静态变量 在类初始化阶段赋值
4. static final int a =10 是真正的编译期常量 在类准备阶段赋值
5. static final int a =new Random().netInt() 运行期确定的静态常量 不是字面量 必须在第二阶段 类初始化阶段 同static
6. static final Object f =new Object() 静态常量引用 还是同5 在第二阶段





## 解析

解析阶段时把符号换成地址。能在解析阶段就直接搞定的，要求是编译期就可以确定，运行期完全不会变。

1. 重载：重载是看传入参数的类型和个数 编译器在编译时就知道调用的是哪个方法 直接在这个阶段替换好地址
2. 私有方法：别人访问不到 之类也不能重写 地址定死了
3. 静态方法 属于类的 地址定死
4. 构造方法 只能调用确定的一个



解析处理不了 必须运行时确定

1. 动态分派：

多态，与实例化的对象有关。必须在运行的时候，顺着队内存里的真实对象，去找对象头里的虚方法表，找出真正的方法地址。
