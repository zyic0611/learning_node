# 事务管理与失效

## 事务传播

也就是方法A调用方法B A开启了事务 B也要开启事务

- REQUIRED 默认设置
  - 逻辑：如果当前没有事务 B就新建一个 如果有事务 就加入这个事务 成为同一个事务
  - 但是只要B发生错误了 A也会跟着一起回滚
- REQUIRES_NEW
  - 逻辑：强制新建一个全新的事务
  - 比如 记录操作日志记录 希望不论成功 日志都要记录 就在记录日子的方法+@Transactional(propagation = Propagation.REQUIRES_NEW)    则不管外面主事务回滚不 日志的事务都会被独立提交

## 事务失效

1. 方法不是public。 springaop扫描的时候默认忽略非public方法 
2. 异常被 catch块吞没了 没有throw出去 事务不会回滚
3. spring默认只遇到运行时异常和error才回滚 受检异常和自定义异常 不会回滚。解法:`@Transactional(rollbackFor = Exception.class)`。
4. 数据库引擎不支持事务 比如MYSQL老版本引擎 MYISAM 现在InnoDB支持回滚
5. 同类方法调用事务 事务失效 绕过了代理。解法： 1.抽离到另一个类 2. 自己注入自己 3. 使用AopContext.currentProxy 获取代理对象。
