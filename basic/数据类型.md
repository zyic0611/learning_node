隐式转换：当小范围向大范围转换的时候 java会自动转换 比如long a =10 是安全的

显式转换：大范围转换为小范围 会带来进度丢失 不安全 



## bigDecimal>double

计算机底层采用二进制存储 所以double也是二进制存储  采用了IEE754标准 所以存的其实是近似值 不是准确值。

如果用 `double` 算钱，`0.1 + 0.2` 算出来的结果很可能是 `0.30000000000000004`

涉及到转账类业务 不能有一丝一毫的差错 所以使用bigDecimal 逻辑是把小数*n倍转换为整数进行运算



## 为什么有了包装类Integer 还要有int

- java是面向对象的 基本数据类型不是对象 java的泛型只支持对象 为了让整型融入面向对象体系 就得有包装类
- 为什么要用基本数据类型： 为了极致的性能和内存占用 int是基本数据类型 运行的时候在虚拟机栈的局部变量里 操作速度快 并且只占4字节 。Integer是个对象 要在堆内存里分配 除了存数据 还要存十几个字节的对象头 如果所有数字都用对象 会占用内存 还会频繁触发GC 性能低



## IntegerCache

机制：类加载的时候 java会默认把-128 127内的所有整型数字提前实例化好 放在堆内存的缓存数组中

是为了缓解Integer频繁创建对象带来的内存消耗 比如for循环

举例：Integer a =100 触发了自动装箱。如果数字在-128~127 会从缓存里直接拿对象用 超过了这个范围 才回去new对象

```Integer a = 100, b = 100; System.out.println(a == b); // true，都指向缓存里的同一个对象 ```

```Integer c = 200, d = 200; System.out.println(c == d); // false，超出缓存范围，各自在堆里 new 了新对象，地址不同```



```java
Integer a = new Integer(100);
Integer b = 100;
Integer c = 100;
int d = 100;

System.out.println(a == b); 
System.out.println(b == c); 
System.out.println(b == d);```
```

a显式调用new 不会走缓存。所以a是一个对象地址。

b c都触发了自动装箱 并且范围在-128~127 所以都指向缓存 d是基本数据类型 在虚拟机栈中

重点！：当==比较一个基本数据类型和包装类的时候 会把包装类自动拆箱 进行纯数值比较。

所以a!=b b==c b==d
