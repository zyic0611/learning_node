# 异常

所有异常和错误都有一个顶级父类 Throwable类 只有他的子类 可以被throw 或者 catch捕获

## 分类

- error 错误 是jvm级别的问题 比如outofmemoryError stackoverflowError  堆溢出 栈溢出 error只能去查jvm或者代码
- exception 程序级别的异常 可以通过代码预防和补救
  - 受检异常 编译时异常 javac强迫开发者处理 不加try catch或者throws 直接报错 这种异常通常认为是外部环境引起的【网络问题 文件丢失 java认为程序需要准备好plan b
  - 非受检异常 运行时异常 代表就是RuntimeException以及所有子类 比如空指针 数组越界 /0  这个纯粹是因为写代码不严谨 不应该用trycatch去掩盖空指针问题 而使用if判断解决

## 关键字

1. throws 写在方法签名上 代表我这个方法可能会出问题 但是不处理 谁调用方法负责处理 是一种可能性声明 可以写多个异常签名
2. throw 方法内部 直接抛出异常实例



## 细节

如果try里有return 执行到return之前 会先去执行finally 再return 

如果try里和finally都有return 则会由finally决定。

所以实际开发中 不会再finally写return
