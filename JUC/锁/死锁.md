# 死锁

## 产生的条件

1. 资源互斥：同一时间只能被一个线程独占
2. 持有并等待：线程拿着旧资源 去申请新的资源 并且不释放资源
3. 不可剥夺：资源只能被持有他的线程释放 外部无法抢夺
4. 环路等待 线程直接形成首尾相接的循环等待链

## 预防死锁

破坏四大条件之一，但是资源互斥无法被破坏。

1. 破坏2 要求线程开始执行时，一次申请所有需要的资源
2. 破坏3 如果线程在一定时间内申请新资源失败，主动释放自己所持有的所有旧资源
3. 破坏4 最常用！ 对资源编号，强制所有线程必须按照编号递增的顺序申请锁。



## 死锁code

public class DeadLockDemo {
    // 1. 准备两个资源（两把锁）
    private static final Object lockA = new Object();
    private static final Object lockB = new Object();

```java
public static void main(String[] args) {
    // 2. 线程 1：先抢 A，再抢 B
    new Thread(() -> {
        synchronized (lockA) {
            System.out.println(Thread.currentThread().getName() + " 拿到了锁A，准备抢锁B...");
            try {
                // 睡 100 毫秒，极其关键！确保线程 2 有时间去把锁 B 抢走
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            
            // 此时 lockB 已经被线程 2 拿走了，这里会无限阻塞！
            synchronized (lockB) {
                System.out.println(Thread.currentThread().getName() + " 拿到了锁B！");
            }
        }
    }, "线程1").start();

    // 3. 线程 2：先抢 B，再抢 A
    new Thread(() -> {
        synchronized (lockB) {
            System.out.println(Thread.currentThread().getName() + " 拿到了锁B，准备抢锁A...");
            try {
                // 睡 100 毫秒，确保线程 1 有时间拿到锁 A
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            
            // 此时 lockA 已经被线程 1 拿走了，这里也会无限阻塞！
            synchronized (lockA) {
                System.out.println(Thread.currentThread().getName() + " 拿到了锁A！");
            }
        }
    }, "线程2").start();
}
```
