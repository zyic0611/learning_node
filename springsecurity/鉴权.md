# 鉴权

**`sys_user` (用户表)**：存账号、密码等基本信息。

**`sys_role` (角色表)**：存角色名称、角色字符（如 `manager`, `admin`）。

**`sys_menu` (菜单表)**：其实是**权限表**。不仅存左侧菜单的路由，更重要的是存按钮级别的**权限标识**（如 `sanitation:alert:query`）。

**`sys_user_role` (用户-角色关联表)**：记录哪个用户拥有哪些角色（多对多）。

**`sys_role_menu` (角色-菜单关联表)**：记录哪个角色拥有哪些权限（多对多）。





### 阶段 1：登录过程（加载权限，生成 Token）

1. **提交凭证**：前端调用 `/login` 接口，传入用户名和密码。
2. **触发认证**：`SysLoginService` 调用 Spring Security 的 `AuthenticationManager.authenticate()` 方法进行认证。
3. **查库比对 (重点)**：底层会自动调用 RuoYi 实现的 `UserDetailsServiceImpl.loadUserByUsername()`。在这个方法里：
   - 根据用户名查 `sys_user` 校验密码。
   - 关联查 RBAC 表，获取该用户所有的**角色集合**和**权限标识集合**（也就是你在前端配的 `sanitation:alert:query` 等）。
   - 将这些信息封装成一个 `LoginUser` 对象。
4. **签发 JWT 并缓存**：认证成功后，RuoYi 生成一个 UUID 作为 Token 的载荷签发 JWT 返回给前端；同时，把 UUID 作为 Key，**`LoginUser` 对象（包含所有权限字符串）序列化为 JSON 存入 Redis**。

### 阶段 2：请求拦截（解析 Token，权限校验）

1. **携带 Token 请求**：前端请求 `/sanitation/alert/list`，Header 带上 JWT。
2. **JWT 过滤器拦截**：请求首先进入 `JwtAuthenticationTokenFilter.doFilterInternal()`。
3. **解析与查缓存**：
   - 解析 JWT 获取 UUID。
   - 用 UUID 去 **Redis** 中查出 `LoginUser` 对象（这避免了每次请求都要去查那五张 MySQL 表，极大提升了高并发下的性能）。
4. **存入上下文**：过滤器将查到的 `LoginUser` 包装成一个 `UsernamePasswordAuthenticationToken`，强制塞入 `SecurityContextHolder` 中。此时，Spring Security 已经“认识”了这个请求的主人以及他所有的权限。
5. **AOP 注解拦截**：请求终于到达 Controller 上的 `@PreAuthorize("@ss.hasPermi('sanitation:alert:query')")`。
   - `@ss` 是 RuoYi 自定义的一个 Bean（`PermissionService`）。
   - 它会从 `SecurityContextHolder` 中掏出刚刚放进去的用户权限集合，判断里面有没有 `sanitation:alert:query`。
   - 有则放行，无则抛出 `AccessDeniedException`，最终返回 403 状态码。
